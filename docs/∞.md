## The afterlife

When every CLI agent speaks A2A natively — no adapter needed — Plumb has two options:

1. **Dissolve.** Hand the adapter specs to a standards body. Publish the ledger format as an RFC. Shut down the for-profit entity. The brass bobs become collector's items.

2. **Evolve.** Become the protocol compatibility layer — the thing that certifies agents are A2A-compliant. A conformance testing service. A UL mark for agent interoperability.

Plumb is designed for option 1. The code has no vendor lock-in. The contract is open. The tests are public. The company is not the product.

## The horizon

The name Plumb means "a weight on a string that defines true vertical."

The horizon is not a feature. It is a direction. Plumb moves toward simpler adapters, more fixtures, fewer surprises. The direction does not change.

The horizon is reached when:
- A new CLI agent ships → someone writes 1 adapter file + 5 fixtures
- An orchestrator wants to delegate → it POSTs to Plumb's A2A endpoint
- An operator wants to debug → they query the ledger with `jq`

That's it. That's the entire job. Plumb does it once, forever.
